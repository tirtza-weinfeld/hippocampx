// ## Table Descriptions

// ### `words`
// - **Primary entity** storing basic word information
// - `word`: The actual word (unique constraint)
// - `pronunciation`: Phonetic pronunciation guide
// - `etymology`: Word origin/history (optional)
// - `frequency_rank`: How common the word is (for sorting/prioritizing)

// ### `word_definitions`
// - **Core content** - stores all definitions for words
// - Each word can have multiple parts of speech (noun, verb, etc.)
// - Each part of speech can have multiple numbered definitions
// - `similar_words` & `opposite_words`: JSON arrays for related terms
// - `example`: Usage example for each definition

// ### `word_relations`
// - **Many-to-many relationship** between words
// - Tracks synonyms, antonyms, and related words
// - `strength`: Relationship strength (1-10 scale)
// - Allows for bidirectional word relationships

// ## Key Design Features

// 1. **Normalized Structure**: Separates word metadata from definitions
// 2. **Flexible Definitions**: Multiple parts of speech per word, multiple definitions per part of speech
// 3. **Rich Relationships**: Dedicated table for word relationships with metadata
// 4. **Performance**: Indexes on frequently queried fields
// 5. **JSON Storage**: Similar/opposite words stored as JSON arrays for flexibility
// 6. **Audit Trail**: Created/updated timestamps on all tables
