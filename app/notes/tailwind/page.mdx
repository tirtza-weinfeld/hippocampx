# Tailwind CSS v4.1: The Complete Guide
## Resizable Table Of Contents

> **Major Paradigm Shift**: Tailwind v4.1 moves from JavaScript configuration to pure CSS with `@theme`, delivering 100x faster builds and built-in container queries.

## Why v4.1 Changes Everything

Tailwind v4.1 represents the most significant architectural change since the framework's inception. The shift to CSS-first configuration eliminates JavaScript complexity while unlocking native browser performance and modern CSS features.

**Core Improvements:**
- **Zero Configuration** - No `tailwind.config.js` needed in most projects
- **CSS-Native Design Tokens** - Configure everything with `@theme` directive
- **Microsecond Rebuilds** - 100x faster incremental builds
- **Built-in Container Queries** - True component-level responsiveness
- **Modern CSS Features** - Text shadows, masks, OKLCH colors, device targeting

## Getting Started

### Installation

Install Tailwind CSS v4.1 with your package manager:

```bash
pnpm add tailwindcss @tailwindcss/postcss
# or
npm install tailwindcss @tailwindcss/postcss
```

### Basic Configuration

Create your `globals.css` with the new CSS-first approach:

```css
@import "tailwindcss";

@theme {
  /* OKLCH Colors - Wide gamut support */
  --color-brand-50: oklch(0.99 0.01 220);
  --color-brand-500: oklch(0.64 0.24 220);
  --color-brand-900: oklch(0.25 0.15 220);
  
  /* Typography Scale */
  --font-display: "Inter", system-ui, sans-serif;
  --text-hero: clamp(2rem, 8vw, 6rem);
  
  /* Custom Spacing */
  --spacing-tight: 0.125rem;
  --spacing-loose: 0.375rem;
  
  /* Extended Breakpoints */
  --breakpoint-xs: 475px;
  --breakpoint-3xl: 1920px;
}

@layer base {
  html { color-scheme: light dark; }
  body { font-family: var(--font-sans); }
  h1, h2, h3, h4, h5, h6 { font-family: var(--font-display); }
}

@layer components {
  .btn {
    @apply px-4 py-2 rounded-lg font-medium transition-colors;
  }
  
  .btn-primary {
    @apply bg-brand-500 text-white hover:bg-brand-600;
  }
}

@layer utilities {
  .glass {
    backdrop-filter: blur(12px);
    background: rgba(255, 255, 255, 0.1);
  }
}
```

## Understanding Theme Variables vs Custom Utilities

This is one of the most important concepts in v4.1. There are **two ways** to create utilities:

### 1. Theme Variables (Auto-Generated Utilities)

Theme variables automatically create utilities based on **naming conventions**. Tailwind recognizes specific prefixes and generates utilities for you:

```css
@theme {
  /* COLOR: --color-* creates bg-*, text-*, border-* utilities */
  --color-primary: oklch(0.64 0.24 220);
  /* ✅ Auto-creates: bg-primary, text-primary, border-primary */
  
  /* FONT: --font-* creates font-* utilities */
  --font-display: "Inter", sans-serif;
  /* ✅ Auto-creates: font-display */
  
  /* SPACING: --spacing-* creates p-*, m-*, w-*, h-* utilities */
  --spacing-tight: 0.125rem;
  /* ✅ Auto-creates: p-tight, m-tight, w-tight, h-tight, etc. */
  
  /* TEXT: --text-* creates text-* utilities for font-size */
  --text-hero: clamp(2rem, 8vw, 6rem);
  /* ✅ Auto-creates: text-hero */
  
  /* SHADOW: --shadow-* creates shadow-* utilities */
  --shadow-glow: 0 0 20px var(--color-primary);
  /* ✅ Auto-creates: shadow-glow */
}
```

**Usage in HTML:**
```html
<div class="bg-primary text-white p-tight shadow-glow">
  <h1 class="font-display text-hero">Hello World</h1>
</div>
```

### 2. Custom Utilities (Manual Creation)

When you need utilities that **don't fit** the theme variable patterns, you create them manually with `@utility`:

```css
/* ❌ Theme variables CAN'T do this - multiple CSS properties */
@utility glass {
    backdrop-filter: blur(12px) saturate(190%);
    background: rgba(255, 255, 255, 0.09);
    border: 1px solid rgba(255, 255, 255, 0.2);
  box-shadow: 0 8px 32px rgba(31, 38, 135, 0.37);
}

/* ❌ Theme variables CAN'T do this - pseudo-selectors */
@utility focus-ring {
  &:focus-visible {
    outline: 2px solid var(--color-blue-500);
    outline-offset: 2px;
    border-radius: 0.25rem;
  }
}

/* ❌ Theme variables CAN'T do this - vendor prefixes */
@utility scrollbar-hidden {
    scrollbar-width: none;
    &::-webkit-scrollbar {
      display: none;
  }
}

/* ❌ Theme variables CAN'T do this - complex selectors */
@utility button-pressed {
  &:active {
    transform: translateY(1px);
    box-shadow: inset 0 2px 4px rgba(0, 0, 0, 0.2);
  }
}

/* ❌ Theme variables CAN'T do this - arbitrary value selectors */
@utility first-child-bold {
  & > *:first-child {
    font-weight: 600;
    color: var(--color-red-500);
  }
}
```

**Usage in HTML:**
```html
<div class="glass">Glass morphism effect</div>
<button class="focus-ring button-pressed">Interactive button</button>
<div class="scrollbar-hidden">Hidden scrollbar</div>
<div class="first-child-bold">
  <p>This first paragraph will be bold and red</p>
  <p>This second paragraph stays normal</p>
</div>
```

**Alternative: Use Tailwind's arbitrary value selectors directly in HTML:**
```html
<div class="[&>*:first-child]:font-semibold [&>*:first-child]:text-red-500">
  <p>This first child becomes bold and red</p>
  <p>This stays normal styling</p>
</div>
```

## Regular Pseudo-Class Syntax vs Arbitrary Value Selectors

### When Regular `first:` and `last:` Syntax Works

Regular Tailwind pseudo-class utilities work when you're styling **the element itself** based on its position relative to siblings:

```html
<!-- ✅ WORKS: Styling the <li> elements themselves -->
<ul>
  <li class="flex py-4 first:pt-5 first:border-t last:pb-5 last:border-b border-red-500">
    <div class="ml-3">
      <p>John Doe</p>
      <p>john@example.com</p>
    </div>
  </li>
  <li class="flex py-4 first:pt-5 first:border-t last:pb-5 last:border-b border-red-500">
    <div class="ml-3">
      <p>Jane Smith</p>
      <p>jane@example.com</p>
    </div>
  </li>
</ul>
```

**Why this works:**
- `first:pt-5` applies to the **first** `<li>` element among its siblings
- `last:pb-5` applies to the **last** `<li>` element among its siblings  
- You're styling the container (`<li>`) based on its position

### When You Need Arbitrary Value Selectors

Use `[&>*:first-child]:` when you want to style **child elements** inside a container:

```html
<!-- ✅ WORKS: Styling children inside the container -->
<div class="[&>*:first-child]:font-semibold [&>*:first-child]:text-red-500">
  <p>This first paragraph becomes bold and red</p>
  <p>This second paragraph stays normal</p>
  <p>This third paragraph stays normal</p>
</div>
```

**Why you need arbitrary syntax here:**
- You want to style the **first child** (`<p>`) **inside** the `<div>`
- Regular `first:` would try to style the `<div>` itself, not its children
- `[&>*:first-child]:` means "target the first child element of this container"

### Key Rule: Target vs Container

**Regular syntax (`first:`, `last:`):**
- Styles **the element itself** based on its position among siblings
- Use when the element you're applying classes to IS the target

**Arbitrary selector syntax (`[&>*:first-child]:`):**
- Styles **child elements** based on selectors
- Use when you want to reach "into" the element and style its contents

### Real-World Examples

```html
<!-- ✅ Style navigation items themselves -->
<nav>
  <a class="px-4 py-2 first:rounded-l-lg last:rounded-r-lg">Home</a>
  <a class="px-4 py-2 first:rounded-l-lg last:rounded-r-lg">About</a>
  <a class="px-4 py-2 first:rounded-l-lg last:rounded-r-lg">Contact</a>
</nav>

<!-- ✅ Style content inside cards -->
<div class="[&>h2:first-child]:mt-0 [&>p:last-child]:mb-0">
  <h2>Card Title</h2>
  <p>Some content here.</p>
  <p>Last paragraph with no bottom margin.</p>
</div>

<!-- ✅ Style table rows themselves -->
<table>
  <tr class="first:border-t-2 last:border-b-2">
    <td>Data 1</td>
  </tr>
  <tr class="first:border-t-2 last:border-b-2">
    <td>Data 2</td>
  </tr>
</table>
```

### When to Use Each Approach

**Use Theme Variables When:**
- ✅ You need **design tokens** (colors, fonts, spacing, shadows)
- ✅ You want **multiple related utilities** (bg-*, text-*, border-* for colors)
- ✅ You're defining **design system values**
- ✅ You want **CSS variables** accessible elsewhere

**Use Custom Utilities When:**
- ✅ You need **multiple CSS properties** in one utility
- ✅ You need **pseudo-selectors** (`:hover`, `:focus`, `:active`)
- ✅ You need **vendor prefixes** or browser-specific code
- ✅ You're creating **component-like patterns**
- ✅ The utility **doesn't fit** theme variable naming conventions
- ✅ You need **arbitrary value selectors** like `[&>*:first-child]:font-semibold [&>*:first-child]:text-red-500`

### Real-World Example

```css
@theme {
  /* Design tokens - auto-generate utilities */
  --color-brand-500: oklch(0.64 0.24 220);
  --spacing-card: 1.5rem;
  --radius-card: 0.75rem;
  --shadow-card: 0 4px 6px -1px rgb(0 0 0 / 0.1);
}

/* Complex pattern - needs custom utility */
@utility card {
  background: var(--color-white);
  padding: var(--spacing-card);
  border-radius: var(--radius-card);
  box-shadow: var(--shadow-card);
  border: 1px solid var(--color-gray-200);
  transition: all 0.2s ease;
  
  &:hover {
    box-shadow: 0 10px 15px -3px rgb(0 0 0 / 0.1);
    transform: translateY(-2px);
  }
}
```

**HTML Usage:**
```html
<!-- Mix of both approaches -->
<div class="card bg-brand-500 text-white">
  <!-- 'card' = custom utility -->
  <!-- 'bg-brand-500', 'text-white' = theme variable utilities -->
</div>
```

This way you get the best of both worlds: **automatic utilities for design tokens** and **manual utilities for complex patterns**!

### Gradients: The Correct Approach

**❌ Wrong:** Don't use `--color-*` for gradients

```css
  /* This WON'T work - gradients can't auto-generate text-*, border-* */
  --color-brand: linear-gradient(45deg, #3b82f6, #8b5cf6);
}
```

**✅ Correct:** Use Tailwind's built-in gradient system (recommended)
```html

<div class="bg-radial-[at_25%_25%] from-white to-zinc-900">Radial gradient</div>
```

**✅ For text gradients:** Use custom utilities
```css
@utility text-gradient-brand {
  background: linear-gradient(45deg, var(--color-blue-600), var(--color-purple-600));
  background-clip: text;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

## Container Queries: Component-Level Responsiveness

One of v4.1's most powerful features is built-in container queries, enabling true component-level responsive design.

### Basic Container Query Usage

```html
<!-- Step 1: Mark element as container -->
<div class="@container max-w-4xl">
  <!-- Step 2: Use @sm, @md, @lg variants -->
  <div class="grid grid-cols-1 @md:grid-cols-2 @lg:grid-cols-3 gap-4">
    <div class="p-4 @sm:p-6">
      Content adapts to container size, not viewport!
    </div>
  </div>
</div>
```

### Named Containers for Precision

```html
<!-- Sidebar with named container -->
<aside class="@container/sidebar w-64 xl:w-80">
  <nav class="@lg/sidebar:flex @lg/sidebar:flex-col space-y-2">
    <a class="@md/sidebar:text-sm @lg/sidebar:text-base">Dashboard</a>
    <a class="@md/sidebar:text-sm @lg/sidebar:text-base">Settings</a>
  </nav>
</aside>

<!-- Main content with different container -->
<main class="@container/content flex-1">
  <article class="@xl/content:prose-xl @lg/content:prose-lg prose">
    <h1>Article adapts to available content space</h1>
  </article>
</main>
```

### When to Use Container vs Viewport Queries

**Use Container Queries For:**
- Component-level responsiveness
- Reusable components that adapt to their space
- Sidebar navigation, cards, modals
- Content that needs to work in various contexts

**Use Viewport Queries For:**
- Overall layout structure
- Global navigation changes
- Page-level responsive behavior

### Advanced CSS Masking

CSS masking allows you to selectively hide portions of an element using images or gradients. Tailwind's masking utilities provide composable controls that mirror the background API but are specifically designed for layering effects.

**What is CSS Masking?**
- **Purpose**: Hide or reveal parts of elements using gradients or images
- **Use cases**: Fade effects, complex reveals, image compositions
- **Advantage**: GPU-accelerated, works with any element content

**Core Mask Types:**

**1. Directional Linear Masks**
```html
<!-- Single-side fade effects -->
<div class="mask-t-from-50% bg-[url(/img/mountains.jpg)]">
  Top fade starting at 50%
</div>

<div class="mask-r-from-30% bg-[url(/img/mountains.jpg)]">
  Right fade starting at 30%
</div>

<div class="mask-b-from-20% mask-b-to-80% bg-[url(/img/mountains.jpg)]">
  Bottom fade range: 20% to 80%
</div>

<div class="mask-l-from-50% mask-l-to-90% bg-[url(/img/mountains.jpg)]">
  Left fade range with precise control
</div>
```

**2. Radial Masks**
```html
<!-- Circular reveal effects -->
<div class="mask-radial-from-80% bg-[url(/img/keyboard.png)]">
  Circular fade from 80% radius
</div>

<!-- Positioned radial masks -->
<div class="mask-radial-at-top-left mask-radial-from-40% mask-radial-to-70% bg-[url(/img/hero.jpg)]">
  Radial fade positioned at top-left corner
</div>

<div class="mask-radial-[ellipse_at_center] mask-radial-from-60% bg-[url(/img/product.jpg)]">
  Custom elliptical mask shape
</div>
```

**3. Composable Multi-Layer Masks**
```html
<!-- Combine linear and radial effects -->
<div class="mask-b-from-50% mask-radial-from-80% bg-[url(/img/mountains.jpg)]">
  Bottom linear fade + radial spotlight
</div>

<div class="mask-r-from-80% mask-t-from-80% mask-radial-from-70% mask-radial-to-85% bg-[url(/img/landscape.jpg)]">
  Multi-directional + radial masking
</div>

<!-- Complex artistic effects -->
<div class="mask-linear-45 mask-from-transparent mask-from-20% mask-to-black mask-to-80% mask-radial-at-bottom-right mask-radial-from-60% bg-[url(/img/abstract.jpg)]">
  Diagonal + positioned radial combination
</div>
```

**4. Image-Based Masks**
```html
<!-- Use images as masks -->
<div class="mask-[url(/masks/organic-shape.svg)] bg-linear-to-br from-purple-500 to-pink-500">
  SVG shape mask over gradient
</div>

<div class="mask-[url(/masks/texture.png)] bg-[url(/img/photo.jpg)]">
  Texture-based reveal effect
</div>
```

**Key Masking Principles:**

**Layering Logic:**
- Multiple masks multiply together (intersection)
- `transparent` areas hide content
- `black`/`white` areas reveal content
- Gradients create smooth transitions

**Performance Tips:**
- Use `will-change: mask` for animated masks
- Prefer CSS gradients over image masks when possible
- Test across browsers (Safari requires `-webkit-` prefixes)

**Common Patterns:**

**Hero Image Fade-Out:**
```html
<section class="mask-b-from-70% bg-[url(/hero.jpg)] bg-cover min-h-screen">
  <div class="pt-20 text-center text-white">
    <h1>Content fades naturally into page</h1>
  </div>
</section>
```

**Card Hover Reveal:**
```html
<div class="group relative overflow-hidden">
  <div class="mask-radial-from-0% group-hover:mask-radial-from-100% bg-[url(/product.jpg)] transition-all duration-700">
    <!-- Image reveals on hover -->
  </div>
</div>
```

**Text Over Image Integration:**
```html

  <div class="mask-t-from-60% mask-b-from-40% bg-[url(/background.jpg)]"></div>

  </div>
</div>
```

## Migration from v3 to v4

### Automated Migration

Tailwind provides an automated upgrade tool:

```bash
npx @tailwindcss/upgrade
```

This tool handles most breaking changes automatically, but manual review is recommended.

### Key Breaking Changes

**Configuration:**
- Replace `@tailwind` directives with `@import "tailwindcss"`
- Move configuration from `tailwind.config.js` to CSS `@theme`
- No `content` array needed - automatic detection

**Utility Changes:**
- `bg-opacity-*` → opacity modifiers (`bg-black/50`)
- `shadow-sm` → `shadow-xs`, `shadow` → `shadow-sm`
- `bg-gradient-*` → `bg-linear-*`

**Example Migration:**

```css
/* v3 Syntax */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* v4 Syntax */
@import "tailwindcss";

@theme {
  --color-primary: #3b82f6;
}
```

## Performance & Best Practices

### Performance Improvements

v4.1 delivers unprecedented build performance:

- **Full builds:** 5x faster than v3
- **Incremental builds:** 100x+ faster (microseconds)
- **Bundle size:** Smaller, optimized CSS output
- **Memory usage:** Significantly reduced
- **Browser performance:** Native CSS features

### Development Best Practices

**✅ Optimal Patterns:**

**CSS-First Configuration:**
```css
@theme {
  --color-brand: oklch(0.64 0.24 220);
  --font-display: "Inter", sans-serif;
}
```

**Layer Organization:**
```css
@layer base { /* Global HTML styling */ }
@layer components { /* Reusable component patterns */ }
@layer utilities { /* Custom single-purpose utilities */ }
```

**Container Queries:**
```html
<div class="@container">
  <div class="@md:flex @md:items-center">
    <!-- Component-level responsiveness -->
  </div>
</div>
```

**Prefer Built-in Features:**
- Use Tailwind's gradient utilities over custom CSS
- Leverage text shadows, masks, and device targeting
- Use theme variables for design tokens, `@utility` for complex patterns

**❌ Anti-Patterns to Avoid:**

- Using `--color-*` for gradients (use built-in gradient utilities instead)
- Mixing v3 and v4 syntax  
- Bypassing the layer system with `!important`
- Using viewport queries when container queries are more appropriate
- Creating custom utilities when theme variables would work
- Ignoring accessibility considerations for motion and input device variants
